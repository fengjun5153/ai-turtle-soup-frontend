# 📜 项目文档：AI 海龟汤——“真相迷雾” (The Fog of Truth)

## 1. 产品定位
一款基于大语言模型（LLM）驱动的**沉浸式逻辑推理游戏**。玩家通过与 AI 扮演的“守密人”进行“是/否”问答，拨开迷雾，还原猎奇或温馨的故事真相。

---

## 2. 核心游戏循环 (The Core Loop)

1.  **开局**：选择汤面（故事片段）。
2.  **推理**：玩家输入问题（文字或语音）。
3.  **反馈**：AI 严格判定“是”、“否”或“无关”，并更新“迷雾浓度”。
4.  **复盘**：玩家点击“结案”，由 AI 评估推理还原度并生成评价。

---

## 3. 核心功能模块 (MVP+)

### 3.1 智能主持人系统 (AI Keeper)
* **三重判定逻辑**：AI 需同时处理玩家提问的**逻辑性**、**事实性**与**合规性**。
* **关键线索追踪**：系统自动识别提问中的关键词（如：身份、动机、死因），当玩家触及核心线索时，UI 给予微弱反馈。
* **理智值 (Sanity) 机制**：
    * 初始 20 点，每次有效提问 -1。
    * 理智值过低时，AI 的回复会变得诡异或模糊。
    * **目的**：防止玩家通过穷举法作弊。

### 3.2 沉浸式交互界面
* **迷雾系统 (The Mist)**：游戏背景采用动态迷雾效果。随着提问次数增加且接近真相，迷雾逐渐稀薄，隐约露出代表真相的插画。
* **证据墙 (Evidence Wall)**：当 AI 回答为“是”时，该问题自动被记录在侧边栏的“证据清单”中，方便玩家串联逻辑。
* **多语言切换 (Immersion Mode)**：
    * 支持**中英双语**一键切换。
    * 玩家可以使用英文提问，AI 以英文回答，并在结算页提供生词库（针对想在游戏中提升语感的玩家）。

### 3.3 结算与复盘
* **真相契合度算法**：采用向量相似度匹配（Semantic Similarity），对比玩家的“最终结案陈词”与“汤底”，给出 0-100 的评分。
* **成就勋章**：如“逻辑奇才（提问次数少）”、“脑洞黑洞（方向完全偏离）”。

## 4. 详细页面规格

| 页面 | 关键功能点 | 视觉/体验要求 |
| :--- | :--- | :--- |
| **大厅 (Library)** | 剧本筛选（难度、题材、时长）、进度保存标识。 | 极简暗黑风，卡片悬停有微光效果。 |

| **揭晓 (Archive)** | 全文朗读（TTS）、推理曲线图、社交分享图。 | 揭晓真相时伴随碎屏或开门声效。 |

## 5. 技术架构建议

### 5.1 提示词工程 (Prompt Engineering)
这是游戏灵魂。建议采用 **System Prompt + Context Window** 隔离法：
> **System**: 你是“守密人”。汤底是[XXX]。严禁提及[关键词]。
> **Constraint**: 只能回答：1. 是 2. 否 3. 与此无关。
> **Hint Logic**: 若玩家请求提示，请给出一个与其已掌握证据相关的模糊隐喻。

### 5.2 前后端栈
* **前端**: Next.js (SSR 提升首屏加载速度) + Tailwind CSS + Framer Motion (动效)。
* **后端**: Node.js / Python Fast API。
* **AI 接口**: 
    * **推理**: GPT-4o 或 DeepSeek-V3（高逻辑性）。
    * **绘图**: Imagen 3（生成汤面封面）。
    * **语音**: OpenAI Whisper (输入) + TTS (输出真相)。

---

## 6. 趣味性扩展 (Phase 2)
* **自定义汤面**：玩家输入一段传闻，AI 自动将其重构为符合逻辑的海龟汤结构。
* **多人协作模式**：多位玩家在一个房间内共同提问，线索共享。
* **全球排行榜**：以“最快破案时长”或“最少提问数”进行全球排名。
